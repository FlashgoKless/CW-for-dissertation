# Архитектурная документация MVP умного вокзала ВСМ

Проект описывает MVP цифровой платформы умного вокзала высокоскоростной магистрали. Платформа проектируется как IT-ядро для оркестрации пассажирского сценария: она связывает билет, рейс, актуальное расписание, карту вокзала, навигацию и подсказки для внешних пользовательских каналов.

Цель MVP - показать, как из разрозненных сервисов вокзала можно собрать управляемый и проверяемый пассажирский путь без попытки заменить все существующие системы.

## Основное решение

- Платформа является оркестратором, а не отдельным мобильным приложением и не единой системой для всех процессов вокзала.
- Пользовательские интерфейсы находятся во внешних каналах: мобильное приложение, сайт, киоск, табло или рабочее место сотрудника.
- Персонализация в MVP контекстная: по рейсу, времени, платформе, карте вокзала и состоянию сессии.
- Полный профиль пассажира, история поездок и документы пассажира не хранятся в платформе.
- Навигация строится по статической карте-графу зон вокзала и точек интереса.

## MVP

В MVP входят:

- создание анонимной сессии пассажирского пути;
- привязка сессии к билету и рейсу через внешнюю билетную систему;
- получение актуального статуса рейса через внешний сервис расписания;
- расчет маршрута по вокзалу по карте-графу;
- выдача подсказок и уведомлений через активную сессию;
- обработка событий изменения рейса и смены платформы.

В MVP не входят:

- полный профиль пассажира;
- хранение истории поездок;
- коммерческие заказы;
- обращения пассажиров;
- багаж и потерянные вещи;
- контроль посадки;
- indoor-позиционирование;
- цифровой двойник вокзала.

## Ключевые сущности

- `JourneySession` - анонимная сессия пассажирского пути.
- `TicketReference` - ссылка на внешний билет без хранения полного билета.
- `TripContext` - актуальный контекст рейса для сессии.
- `StationNode` - зона, вход, платформа, сервисная точка или точка интереса на карте вокзала.
- `RouteSegment` - участок маршрута между узлами карты.
- `ScenarioStep` - текущий шаг пассажирского сценария.
- `Hint` - подсказка или рекомендация для пользовательского канала.
- `NotificationDelivery` - попытка доставки подсказки в канал.
- `ExternalEvent` - внешнее событие с идемпотентной обработкой.
