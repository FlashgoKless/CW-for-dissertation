# ADR 0004: Делать платформу API-first для внешних каналов

## Статус

Accepted.

## Контекст

Пассажир может взаимодействовать с вокзалом через мобильное приложение, сайт, киоск, табло или будущие каналы. Если MVP будет строиться вокруг одного интерфейса, сценарная логика может оказаться привязанной к конкретному каналу.

## Решение

Платформа предоставляет API для создания сессии, чтения состояния, маршрута, подсказок и подтверждения получения подсказки. Пользовательские интерфейсы находятся вне платформы.

## Последствия

Плюсы:

- Разные каналы используют один сценарный слой.
- Можно развивать UI независимо от доменной логики.
- Контракты API становятся основой интеграции.

Минусы:

- Нужно проектировать стабильные контракты с самого начала.
- Для демонстрации может понадобиться отдельный mock-канал.
