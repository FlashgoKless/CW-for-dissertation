# ADR 0003: Использовать карту-граф для навигации

## Статус

Accepted.

## Контекст

Для бесшовного пассажирского пути нужна навигация по вокзалу. Indoor-позиционирование и цифровой двойник дают больше возможностей, но требуют оборудования, точных данных и сложной эксплуатации.

## Решение

В MVP навигация строится по статической карте-графу. Узлы графа описывают входы, зоны, платформы, лифты, эскалаторы, сервисные точки и другие точки интереса. Ребра описывают переходы, время и ограничения доступности.

## Последствия

Плюсы:

- Реализуемо без дополнительной инфраструктуры позиционирования.
- Можно тестировать алгоритм маршрута на фиксированных данных.
- Карта версионируется через `map_version`.

Минусы:

- Пассажир должен выбрать или передать начальную точку.
- Нет автоматического определения текущего положения.
- Точность зависит от качества карты.
