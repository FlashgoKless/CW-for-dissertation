# ADR 0001: Использовать платформу как оркестратор пассажирского сценария

## Статус

Accepted.

## Контекст

Вокзал ВСМ включает несколько самостоятельных сервисов: билеты, расписание, навигацию, услуги, обращения, посадку и служебные системы. Если MVP попытается заменить все эти сервисы, границы станут слишком широкими, а архитектура будет плохо проверяемой.

## Решение

Платформа в MVP является оркестратором пассажирского сценария. Она хранит `JourneySession`, связывает внешние факты с состоянием сценария, рассчитывает маршрут и создает подсказки.

Билеты, расписание, каналы доставки и другие сервисы остаются внешними системами.

## Последствия

Плюсы:

- MVP можно реализовать без замены существующих систем.
- У платформы появляются четкие границы ответственности.
- Новые каналы получают единый API сценария.

Минусы:

- Платформа зависит от качества внешних API и событий.
- Нужны адаптеры, контрактные тесты и стратегия деградации.
