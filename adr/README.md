# ADR

Журнал архитектурных решений для MVP платформы умного вокзала ВСМ.

| ADR | Статус | Решение |
|---|---|---|
| [0001](0001-platform-orchestrator.md) | Accepted | Использовать платформу как оркестратор пассажирского сценария |
| [0002](0002-no-full-passenger-profile.md) | Accepted | Не хранить полный профиль пассажира в MVP |
| [0003](0003-navigation-graph.md) | Accepted | Использовать карту-граф для навигации |
| [0004](0004-api-first-channels.md) | Accepted | Делать платформу API-first для внешних каналов |
| [0005](0005-event-integration-idempotency.md) | Accepted | Обрабатывать внешние события идемпотентно |
